## RFP Section Drafting

When drafting an RFP response section, follow this process:

### 1. Clarify the Section
Ask what RFP section they need (executive summary, technical approach, team, pricing, timeline, references, compliance).

### 2. Gather Inputs
Before writing, confirm:
- Target audience (technical evaluators, procurement, executives)
- Word/page limit if any
- Required subsections from the RFP
- Any mandatory compliance language

### 3. Draft Structure

#### Executive Summary
- Opening hook: the client's challenge in their language
- Proposed solution in 2-3 sentences
- Key differentiator (why us)
- Expected outcome with metric

#### Technical Approach
- Architecture diagram description (agents, data flow, integrations)
- Technology stack table: component | technology | justification
- Security measures relevant to their industry
- Scalability and monitoring approach

#### Delivery Timeline
- Phase table: phase | duration | deliverables | team
- Milestones with go/no-go criteria
- Dependencies and assumptions listed explicitly
- Risk register: risk | probability | impact | mitigation

#### Pricing
- Line items by phase and role
- Rate card with blended rate option
- Fixed-price vs T&M recommendation based on scope clarity
- Optional items separated clearly

### 4. Output Format
Use clean markdown with headers. No fluff paragraphs. Every sentence should earn its place.
Mark assumptions with [ASSUMPTION] and items needing client input with [CLIENT INPUT NEEDED].
